# ADR-006: Política de versiones

## Estado

Aceptado

## Decisión

- El manual usa **semantic versioning** (MAJOR.MINOR.PATCH)
- MAJOR: cambios estructurales mayores o contenido incompatible con versiones anteriores
- MINOR: nuevos capítulos, nuevas secciones, expansiones significativas
- PATCH: correcciones, actualizaciones de snapshot, ajustes menores
- Cada release debe incluir `SOURCE_SNAPSHOT.md` actualizado con los commits congelados
